# Visual Companion Guide

## What This Is

A browser-based companion for showing mockups, diagrams, and visual options
during brainstorming. It is a tool, not a mode. Accepting it means it is
available for questions that benefit from visual treatment — it does NOT mean
every question uses the browser.

## Per-Question Decision Rule

For EACH question, ask: would the user understand this better by SEEING it
than by READING it?

**Use the browser for:**
- UI mockups and wireframes
- Layout comparisons (side-by-side)
- Architecture diagrams
- Visual design options

**Use the terminal for:**
- Requirements questions
- Conceptual tradeoffs (A/B/C text options)
- Scope decisions
- Any question where the answer is words, not shapes

A question about a UI topic is NOT automatically a visual question.
"What does personality mean in this context?" → terminal.
"Which wizard layout works better?" → browser.

## How to Open the Companion

Instruct the user to open the local URL provided by the IDE's browser sub-agent.
Capture a screenshot of each visual presented and reference it in the next
terminal message to maintain context continuity.
